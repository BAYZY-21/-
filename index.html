✎ Bluetooth is short-range (~10m standard, ~50m BLE). Long-range technologies include LoRaWAN (km range) and Cellular (4G/5G).

5 Not every Embedded System is necessarily an IoT system. TRUE
6 "Middleware" is another name for the Network Layer. FALSE
✎ Middleware is another name for the Processing Layer (also called Transport Layer in some models) — not the Network Layer.

Q2True or False — Correct the False Ones
1-F2-T 3-F4-F
1 Microprocessors (MPUs) contain built-in Flash memory and RAM on the same chip, making them ideal for small, low-power IoT sensors. FALSE
✎ That describes Microcontrollers (MCUs). MPUs are high-performance processors requiring external memory (RAM/ROM) — used for complex tasks like video or OS-based processing.
2 The D2G model is necessary when IoT devices use local protocols like Zigbee that cannot connect directly to the global Internet. TRUE
3 In the D2D communication model, IoT devices must connect to the global Internet to exchange data with each other. FALSE
✎ In D2D, devices communicate locally and directly (e.g., via Bluetooth or Zigbee) without needing the Internet.
4 GPIO pins on a microcontroller can only be configured as input pins to read data from sensors. FALSE
✎ GPIO pins can be configured as Input (read sensors) OR Output (control actuators like LEDs, motors, relays).
Q3Choose the Correct Answer
Q3.1
Best connectivity for an underground smart water meter sending a few bytes once a day?
a) Wi-Fi
b) Bluetooth
c) LoRaWAN
d) Cellular (5G)
LoRaWAN = Long Range + Very Low Power — perfect for rare, small-data transmissions over large distances underground
Q3.2
Connect a sensor to MCU using only 2 wires with a clock signal — best interface?
a) SPI
b) UART
c) I2C
d) GPIO
I2C uses 2 wires (SDA+SCL) and is synchronous (has clock). SPI needs 4 wires. UART uses 2 wires but is asynchronous (no clock).
Q3.3
Publishing sensor data to multiple subscribers simultaneously with low bandwidth overhead?
a) HTTP
b) MQTT
c) CoAP
d) TCP/IP
MQTT Publish/Subscribe: one publisher sends to a broker which distributes to ALL subscribers. Very lightweight protocol.
Q4Design Scenario — Smart Agriculture Farm (5 km radius)
📋 Scenario
Design an IoT system to monitor soil moisture over a large farm (5 km radius). Sensors run on small batteries that must last for years.
تصميم نظام إنترنت الأشياء لمزرعة ذكية لمراقبة رطوبة التربة على مساحة نصف قطرها 5 كيلومترات — المستشعرات تعمل على بطاريات صغيرة تدوم لسنوات.
Question 1 — Processing Unit
MCU or MPU? Justify.
✓ MCU (Microcontroller)
Very low power consumption + built-in memory + simple task (reading soil moisture) = no need for MPU's high computational power.
Question 2 — Connectivity
Most suitable wireless technology?
✓ LoRaWAN
Long-range (up to km) + extremely low power = ideal for battery-powered sensors across a large farm area.
Question 3 — Protocol
HTTP or MQTT?
✓ MQTT
Lightweight + minimal overhead + Publish/Subscribe model + saves battery. HTTP's request-response cycle is too heavy for constrained devices.

Q2True or False — Correct the False Ones
1-T2-F 3-T4-F
1 I2C communication requires fewer wires than SPI but is generally slower. TRUE
2 In UART communication, the sender and receiver must share a Clock signal to transfer data successfully. FALSE
✎ UART is Asynchronous — it does NOT use a clock signal. Both devices agree on the same baud rate (speed) instead of sharing a physical clock line.
3 HTTP is a heavy protocol with high overhead, making it less suitable for battery-powered IoT devices compared to MQTT. TRUE
4 SPI supports Multi-Master and Multi-Slave using only two wires for communication. FALSE
✎ I2C uses 2 wires (SDA+SCL) and supports Multi-Master. SPI uses 4 wires and typically has a Single-Master with multiple slaves selected via separate SS pins.

Q4Design Scenario — Smart Security Camera (Bank)
📋 Scenario
Design a Smart Security Camera for a bank: capture HD video, process faces in real-time, send video stream directly to a central server.
تصميم نظام كاميرات مراقبة ذكية لبنك: التقاط فيديو عالي الدقة + معالجة الوجوه في الوقت الفعلي + إرسال البث مباشرةً لخادم مركزي.
Question 1 — Processing Unit
MCU or MPU? Justify.
✓ MPU (Microprocessor)
HD video + real-time face recognition = high computational power + large external RAM. Only an MPU (e.g., Raspberry Pi CPU) can handle this. MCUs are too weak.
Question 2 — Connectivity
Best technology for high data transfer?
✓ Wi-Fi or Ethernet
HD video requires high bandwidth. Wi-Fi/Ethernet support Mbps–Gbps speeds. LoRaWAN and Bluetooth are far too slow for continuous video streaming.
Question 3 — Communication Model
D2D or D2C?
✓ Device-to-Cloud (D2C)
Camera sends directly to central server (cloud) via internet — no intermediate gateway. It's a direct Device-to-Cloud connection.

Q2True or False — Correct the False Ones
1-F2-T 3-T4-F
1 In MQTT, IoT devices communicate directly with each other without any central server. FALSE
✎ MQTT requires a central "Broker" server. Publishers send TO the broker; the broker routes messages to all matching subscribers. No direct device-to-device communication.
2 CoAP is designed for constrained IoT devices and operates over UDP to reduce overhead. TRUE
3 Cellular networks (4G/5G) provide long-range communication but consume significantly more power than LoRaWAN. TRUE
4 In an MCU, Flash memory is used for temporary data storage, while RAM permanently stores the program code. FALSE
✎ Exact opposite: Flash memory permanently stores the program code. RAM is used for temporary data storage during execution (lost when powered off).

Q4Design Scenario — Wearable Smartwatch (Hospital Patient Monitoring)
📋 Scenario
A wearable smartwatch sends a patient's heart rate to a smartphone via Bluetooth. The smartphone then connects to the Internet to forward data to the hospital server.
ساعة ذكية ترسل معدل ضربات القلب إلى هاتف ذكي عبر البلوتوث، ثم يُرسل الهاتف البيانات إلى خادم المستشفى عبر الإنترنت.
Question 1 — Communication Model
What model is used? What role does the phone play?
✓ Device-to-Gateway (D2G)
The watch cannot reach the internet directly (uses Bluetooth). The smartphone acts as a Gateway — bridging Bluetooth to the global internet.
Question 2 — Connectivity Watch↔Phone
Best technology for the watch-to-phone link?
✓ Bluetooth / BLE
Short-range personal device communication. BLE (Bluetooth Low Energy) is perfect for wearables — low power, preserving battery life.
Question 3 — Internal Sensor Interface
Connect heart sensor to processing unit with 2 wires — SPI or I2C?
✓ I2C
I2C uses exactly 2 wires (SDA + SCL), supports multiple sensors on one bus. Standard choice for compact wearable sensor integration.

Q1Multiple Choice — Choose Ten Only
1
The ultrasonic sensor HC-SR04 measures:
a) Temperature
b) Humidity
c) Distance
d) Light intensity
HC-SR04 emits ultrasonic pulses; measures echo return time → Distance = (Time × 343 m/s) / 2
2
The Arduino Uno operates at which voltage?
a) 9.3V
b) 1.8V
c) 5V
d) 12V
Arduino Uno = 5V logic. Powered via USB (5V) or barrel jack (7–12V regulated to 5V internally)
3
Which display type uses I2C communication?
a) LCD (direct parallel)
b) Seven-segment
c) OLED
d) TFT
OLED displays (SSD1306 chip) commonly use I2C protocol (address 0x3C). TFT uses SPI. Basic LCD uses parallel or SPI.
4
An LDR sensor produces which type of signal?
a) Digital
b) Analog
c) PWM
d) Binary
LDR = Light Dependent Resistor — passive analog component. Resistance varies with light → produces analog voltage read via ADC pin.
5
In which IoT architecture layer do sensors belong?
a) Application Layer
b) Network Layer
c) Perception Layer
d) Business Layer
Perception Layer = physical foundation — all sensors, actuators, and physical devices reside here.
6
Correct distance formula for ultrasonic sensor?
a) Distance = Time + Speed of Sound
b) Distance = Time × Speed of Sound
c) Distance = (Time × Speed of Sound) / 2
d) Distance = Speed of Sound / Time
Sound travels TO the object AND BACK → divide by 2 for one-way distance. Speed of sound ≈ 343 m/s in air.

MODELSIoT Communication Models — D2D · D2C · D2G

Devices communicate directly — locally, no internet needed.

D2C — Device to Cloud
